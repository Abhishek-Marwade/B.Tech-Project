Research Compass - Complete System Architecture

🎯 System Overview

Research Compass is a Production-Ready Multi-Agent RAG (Retrieval-Augmented Generation) System designed for academic research. It intelligently searches, organizes, summarizes, and answers questions about research papers using a Progressive Knowledge Base architecture.

Core Philosophy

  • Local-First: Check local knowledge base before calling external APIs
  • AI-Powered: Uses Groq (Llama 3.3 70B) for fast, intelligent processing
  • Progressive Learning: Continuously builds knowledge from searches
  • Migration-Ready: SQLite/Local → PostgreSQL/Cloud seamless migration

🧠 Agent System Deep Dive

The system uses 4 specialized AI agents working together:

1. 🔍 Search Agent (app/agents/search_agent.py)

Purpose: Find relevant research papers using Progressive RAG

Workflow:

User Query
    ↓
1. Embed Query (Gemini)
    ↓
2. Search Local Vector Store (Cosine Similarity)
    ↓
3. Found >= 10 papers? → Return Results ✅
    ↓ (if not enough)
4. Fetch from Springer API
    ↓
5. Normalize Papers (Preprocessor)
    ↓
6. Store in Local KB (Papers DB + Vector Store)
    ↓
7. Rank Combined Results (Ranker)
    ↓
8. Return Top 10 Papers

Key Features: - Progressive RAG: Local-first retrieval reduces API calls by ~70% - Similarity Threshold: Default 0.7 (adjustable) - Batch Storage: Stores all new papers for future queries - Incremental Embeddings: Only generates embeddings for new papers

Code Flow:

def search_agent_logic(query, max_results=20, top_k=10):
    # 1. Local vector search
    query_embedding = embed_texts([query])[0]
    local_results = vector_store.search_similar(query_embedding, top_k, threshold=0.7)

    # 2. Check if sufficient
    if len(local_papers) >= top_k:
        return local_papers

    # 3. Fetch from APIs
    api_papers = search_semanticscholar(query, max_results)

    # 4. Store new papers
    new_count, dup_count = store_papers_batch(api_papers)

    # 5. Generate embeddings for new papers
    embeddings = embed_texts(new_paper_texts)
    vector_store.add_embeddings_batch(new_paper_ids, embeddings)

    # 6. Rank and return
    return rank_papers(query, combined_papers, top_k)

2. 📝 Summarizer Agent (app/agents/summarizer.py)

Purpose: Generate academic-quality summaries with technical depth

Input: List of papers (max 5 per batch)

Output: Structured JSON summaries

Structure:

{
  "title": "Paper Title",
  "methodology": "100-120 words: algorithms, models, frameworks, hyperparameters",
  "dataset": "40-60 words: name, size, preprocessing, splits",
  "results": "80-100 words: metrics, baselines, statistical significance",
  "limitations": "40-60 words: weaknesses, constraints, future work"
}

Features: - Smart Caching: Cached by content hash (not query) using SQLite - Token Limit: 6000 tokens (supports detailed analysis) - Fallback Mode: Returns abstracts if AI disabled - Academic Focus: Designed for IIT professor-level review

Prompt Strategy:

You are a senior research analyst preparing detailed reports for IIT professors.

For each paper, provide comprehensive analysis (250-300 words total):
- Methodology: Specific algorithms, architectures, techniques
- Dataset: Name, size, preprocessing details
- Results: Exact metrics (accuracy, F1, RMSE), comparisons
- Limitations: Critical analysis, future work

2. Vector Store (app/database/vector_store.py)

Storage Format:

data/embeddings/
├── vectors.npy       → NumPy array (N x 768)
├── index.json        → {paper_id: vector_index} mapping
└── metadata.json     → {model, dim, count}

Architecture:

class VectorStore:
    vectors: np.ndarray      # Shape: (N, 768)
    index: dict[str, int]    # {paper_id: vector_idx}

    def search_similar(query_embedding, top_k, threshold):
        # Cosine similarity: query · vectors / ||query|| ||vectors||
        similarities = cosine_similarity(query, all_vectors)
        return top_k results where similarity >= threshold

Similarity Search Algorithm:

# 1. Normalize vectors
query_norm = query / ||query||
vectors_norm = vectors / ||vectors||

# 2. Compute cosine similarity
similarities = vectors_norm @ query_norm  # Dot product

# 3. Filter by threshold
valid_idx = where(similarities >= 0.7)

# 4. Sort and return top-k
top_k_idx = argsort(similarities[valid_idx])[-top_k:]
return [(paper_id, score), ...]

🛠️ Supporting Components

Preprocessor (app/agents/preprocess.py)

  • Normalizes paper data from different APIs
  • Ensures consistent schema
  • Handles missing fields gracefully

Ranker (app/agents/ranker.py)

  • Ranks papers by relevance to query
  • Uses simple scoring (can be enhanced with ML)

API Client (app/agents/api_client.py)

  • Current: Springer Metadata API v2
  • Previously: Semantic Scholar (commented code)
  • Handles rate limits, timeouts, errors

Embeddings (app/utils/embeddings.py)

  • Uses Google Gemini models/embedding-001
  • Generates 768-dimensional vectors
  • Batch processing for efficiency

Cache Manager (app/utils/cache.py)

  • SQLite-based caching (cache.db)
  • Keys: content hash + model suffix
  • Reduces duplicate AI calls

AI Config (app/utils/ai_config.py)

  • Unified LLM interface
  • Supports Groq (primary) and Gemini (fallback)
  • Environment-based configuration
